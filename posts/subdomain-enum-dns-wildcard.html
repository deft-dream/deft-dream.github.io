概述

子域名枚举是SRC漏洞挖掘的第一步。通过收集目标的所有子域名，可以扩大攻击面，发现更多潜在漏洞。

猫猫提示：子域名就像猫咪藏起来的玩具，要一个个找出来~

常用工具

1. Subfinder

# 安装
go install -v github.com/projectdiscovery/subfinder/v2/cmd/subfinder@latest

# 使用
subfinder -d target.com -o subdomains.txt

2. Amass

# 被动枚举
amass enum -passive -d target.com -o subs.txt

# 主动枚举（更全面）
amass enum -active -d target.com -o subs_active.txt

3. OneForAll

# 国内很好用的工具
python3 oneforall.py --target target.com run

枚举方法

1. 暴力枚举

# 使用字典
gobuster dns -d target.com -w wordlist.txt -t 50

2. 证书透明度

# crt.sh查询
curl -s "https://crt.sh/?q=%25.target.com&output=json" | jq -r '.[].name_value' | sort -u

3. 搜索引擎

# Google Hacking
site:target.com -www

# Shiro/ZoomEye等

4. 第三方数据

# VirusTotal
curl -s "https://www.virustotal.com/api/v3/domains/target.com/subdomains" \
  -H "x-apikey: YOUR_KEY"

DNS泛解析

有些域名配置了泛解析，任何子域名都会解析到同一个IP，导致枚举结果大量误报。

检测泛解析

# 随机子域名测试
nslookup random123.target.com
nslookup random456.target.com

# 如果都解析到同一IP，说明存在泛解析

处理泛解析

# 方法1：对比响应内容
# 泛解析的页面通常内容相同

# 方法2：使用工具的泛解析过滤
subfinder -d target.com -nW

# 方法3：IP黑名单
# 将泛解析IP加入黑名单，过滤结果

实战技巧

  • 多工具交叉验证，取并集
  • 关注开发/测试环境子域名（dev, test, staging）
  • 检查历史DNS记录（SecurityTrails）
  • 关注邮箱服务器域名（mx记录）
  • 子域名爆破时注意控制速率，避免被封
